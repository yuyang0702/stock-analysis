> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [x]`) syntax for tracking.

**Goal:** Implement verified daily SQLite online backups, 7/4/12 retention, quarterly restore drills, status reporting, alerts, and Linux scheduling without changing trading behavior.

**Architecture:** A single standard-library CLI in `trading_backup.py` reuses `TradingStore.backup_to` and validates every published copy before atomically placing it in daily, weekly, and monthly slots. Manifests under `manifests/<tier>/` are the source of backup metadata; a separate quarterly drill copies one validated backup into an isolated temporary directory and compares integrity, schema, and table counts. `run_ubuntu.sh` installs independent oneshot services and timers; backup failures never alter trading state.

**Tech Stack:** Python 3 standard library, SQLite through existing `TradingStore`, existing `WeComNotifier`, `unittest`, Bash/systemd templates already generated by `run_ubuntu.sh`.

## Global Constraints

- Do not change signal generation, buy/sell rules, risk limits, `KILL_SWITCH`, health gates, or JoinQuant order behavior.
- Do not add third-party dependencies.
- Default backup root is `/opt/stock-analysis-backups`, outside the project directory.
- Retain 7 daily date slots, 4 ISO-week slots, and 12 calendar-month slots.
- A failed run must not replace or delete an existing valid backup.
- The drill must never initialize, migrate, write, or replace the live database.
- Reports and manifests must not contain tokens, webhooks, environment values, or full account details.
- Do not commit, push, deploy, install timers, or restart services without a separate user authorization.

```python
CORE_TABLES = (
    "schema_migrations", "strategy_runs", "signals", "risk_decisions", "system_state",
    "position_cycles", "order_events", "exit_intents", "trade_cooldowns",
    "orders", "fills", "account_snapshots", "position_snapshots", "daily_equity",
    "reconciliation_runs", "reconciliation_items", "control_events",
)

def database_facts(db_file: Path) -> dict[str, object]:
    store = TradingStore(db_file)
    with store.connect() as conn:
        tables = {str(row[0]) for row in conn.execute("SELECT name FROM sqlite_master WHERE type='table'")}
        version_row = conn.execute("SELECT MAX(version) FROM schema_migrations").fetchone()
        counts = {name: int(conn.execute(f'SELECT COUNT(*) FROM "{name}"').fetchone()[0])
                  for name in CORE_TABLES if name in tables}
        check = str(conn.execute("PRAGMA integrity_check").fetchone()[0])
    return {"integrity_check": check, "schema_version": int(version_row[0] or 0), "table_counts": counts}
```
