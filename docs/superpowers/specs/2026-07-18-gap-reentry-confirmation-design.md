## 1. 目标

现有策略要求当前价达到计划入场位，同时通过追高、涨停、流动性、组合风险和 100 股交易单位检查。该边界能够阻止盲目追涨，但会出现另一类可审计的机会：

- 股票已经形成过有效买入计划；
- 后续价格跳空越过计划入场价，原信号没有成交；
- 股票仍被当前时点策略选中；
- 涨停打开后出现真实可交易窗口；
- 当前价格和风险仍允许小仓位试错。

本设计新增“跳空越价后二次确认入场”路径。它不是恢复旧信号，也不是普遍放宽追高限制，而是在当前策略重新验证、开板稳定性确认和完整风险检查之后生成一条全新的买入执行契约。

本能力在实现后直接用于 JoinQuant 模拟盘验证，不设置只展示但不执行的影子交易路径。进入真实资金前仍必须单独授权并重新评估。

## 2. 非目标

本设计不：

- 替换现有候选池、评分、普通突破买入或退出策略；
- 自动追买封死涨停、集合竞价价格或无真实卖盘的订单；
- 恢复、延长或重新发布过期买入信号；
- 绕过 `RISK_OFF`、人工停止买入、kill switch、健康门、对账门或组合风险上限；
- 因为目标仓位不足 100 股而默认把仓位放大到一手；
- 允许同股票重复信号、重复委托或当天反复开板时无限尝试；
- 把模拟盘的少数成功案例直接标记为 `validated`。

## 3. 与现有规则的关系

### 3.1 保持不变

下列规则继续作为硬边界：

- 服务器生成唯一版本化买入执行契约；
- JoinQuant 下单前使用实际价格、现金、持仓、未完成订单和交易状态重新校验；
- `RISK_OFF` 禁止新买，`CAUTION` 使用更高评分门槛和更低风险预算；
- 单票、行业、题材、无分类、总仓位、持仓数量和开放风险上限；
- 停牌、特殊上市阶段、行情陈旧、流动性不足和无效价格拒绝；
- 同股票已有持仓、未完成买单或冷却期时拒绝新买；
- 卖出、硬止损、活动退出意图和风险降级优先于任何新增买入；
- A 股 T+1、涨跌停和 100 股交易单位约束。

### 3.2 有限例外

二次确认路径只允许对以下两项进入专用判断：

1. 当前价高于原计划入场价、会被普通 `buy_chasing` 拒绝；
2. 正常风险定仓向下取整后不足 100 股。

其他风控不得豁免。

## 4. 名词和价格锚点

### 4.1 原计划

原计划必须来自已经落账的版本化买入执行契约，至少包含：

- `parent_signal_id`
- `original_entry_price`
- `original_stop_price`
- `original_risk_r`
- `parameter_version`
- `strategy_version`
- 原计划创建、验证和失效时间

原计划只提供可审计锚点，不能直接执行。

### 4.2 原始风险单位

```text
original_risk_r = original_entry_price - original_stop_price
```

要求 `original_risk_r > 0`。缺少有效原止损或风险单位时，二次确认失败关闭。

### 4.3 二次入场最高价

```text
reentry_cap_price = original_entry_price + 0.5 × original_risk_r
```

该价格是绝对上限，不因评分、涨幅、封板次数或人工预期自动放宽。

例如原入场价 74.72、原止损价 69.49：

```text
1R = 5.23
二次入场最高价 = 77.34
```

76.89 可以进入后续确认；77.34 以上拒绝；实际涨停附近不得买入。

## 5. 资格门

进入二次确认前必须同时满足：

1. 存在有效且可追溯的原计划；
2. 当前五分钟批次重新选中该股票，不能只依赖历史推荐；
3. 当前原策略分达到当前市场状态要求；
4. 当前交易依据、题材、流动性、压力位和风险收益比仍有效；
5. 当前市场不是 `RISK_OFF`；
6. `buy_enabled=1`、`kill_switch=0`，健康、对账和执行门均允许买入；
7. 当前没有该股票持仓、活动买入订单、未知提交、买入冷却或同日已成交二次入场；
8. 当前行情新鲜，证券可交易，涨停价和板块交易规则可确认；
9. 当前价格已越过原计划价但不高于二次入场最高价；
10. 当前不是集合竞价、午间休市或禁止新开仓的尾盘窗口。

任何一项失败都不得创建可执行买入信号。

## 6. 状态机

每只股票每个交易日最多维护一个二次确认机会：

```text
INELIGIBLE
  → GAP_DETECTED
  → LOCKED_LIMIT | OPEN_OBSERVING
  → OPEN_CONFIRMED
  → SIGNAL_PUBLISHED
  → ORDER_SUBMITTED
  → PARTIALLY_FILLED | FILLED | CANCELLED | REJECTED
```

终止状态包括：

```text
TOO_FAR
FALLING
RESEALED
ATTEMPTS_EXHAUSTED
TOO_LATE
RISK_REJECTED
EXPIRED
```

状态转换必须幂等。同一行情批次、同一机会和同一信号重复处理不能增加确认次数或重复发布。

## 7. 涨停、开板与稳定确认

### 7.1 开盘或盘中封死涨停

当股票处于当日实际涨停价且没有真实可成交卖盘：

- 进入 `LOCKED_LIMIT`；
- 不委托、不排队、不发布正式买入信号；
- 允许发送一次“涨停锁定，未追入”的状态通知；
- 后续批次继续观察是否出现真实可交易窗口。

涨停判断必须使用当日证券元数据或权威涨停价，不能把所有股票写死为 10%。

### 7.2 第一次开板

第一次出现低于涨停价的真实可交易报价时：

- 开板尝试次数加一；
- 进入 `OPEN_OBSERVING`；
- 保存第一次观察时间、价格、涨停价和行情批次 ID；
- 此时仍不生成买入信号。

第一次观察必须重新通过资格门和价格上限。

### 7.3 第二次独立扫描确认

至少经过一个新的有效五分钟交易批次后，才允许确认。第二次扫描必须满足：

- 与第一次具有不同的行情批次 ID；
- 有效交易时间至少相隔 5 分钟，午间休市不计入；
- 股票仍未封涨停并存在可交易报价；
- 当前价不高于 `reentry_cap_price`；
- 当前价相对第一次观察价下跌不超过 1%；
- 当前评分、市场状态、流动性、账户和组合风险仍合格；
- 重新计算的止损、仓位、止盈和风险收益比有效。

通过后进入 `OPEN_CONFIRMED`，创建新的信号 ID 和完整执行契约。

### 7.4 回封

确认完成前重新封板：

- 当前观察确认清零；
- 状态记录为 `RESEALED`；
- 不保留待成交信号或委托；
- 再次开板时从第一次观察重新开始。

一天最多允许两次开板观察。第三次开板进入 `ATTEMPTS_EXHAUSTED`，当天不再参与。

### 7.5 执行前回封或涨价

JoinQuant 下单前必须重新检查实际价格。出现以下任一情况时取消：

- 已重新封涨停；
- 实际价格高于 `reentry_cap_price`；
- 行情、现金、账户或组合风险已经变化；
- 出现同股票持仓或未完成订单；
- 信号已过期。

取消后不得在同一信号上等待价格回落，也不得重复下单。

## 8. 时间边界

### 8.1 集合竞价

09:15–09:29 只观察，不参与二次确认，不发布可执行买入信号。第一次有效观察从连续竞价开始。

### 8.2 午间休市

11:29 的第一次观察不能在 13:00 因自然时间经过 91 分钟而直接通过。只计算有效交易分钟，并要求下午出现新的独立扫描。

### 8.3 尾盘

- 14:45 后不再启动新的二次确认；
- 已开始的机会必须在 14:45 前完成第二次确认；
- 正式信号最迟在 14:50 前发布；
- 14:50 后不新增此类仓位。

这是因为尾盘剩余观察和成交时间不足，且 A 股当日新买仓位无法当日止损。

## 9. 新执行计划

二次确认不得机械沿用原止盈、仓位或原信号时间。

```text
new_entry_price = 确认时预计可成交价

new_effective_stop =
max(
  original_stop_price,
  新入场价对应的板块最大亏损边界,
  当前结构/ATR 生成的有效止损
)

new_risk_r = new_entry_price - new_effective_stop
```

要求：

- `new_risk_r > 0`；
- 止损只能相对原计划保持或收紧，不能为容纳追高而放宽；
- 按新入场价重新计算压力位、止盈位和风险收益比；
- 风险收益比低于现行要求时拒绝；
- 成交后继续按实际成交价执行现有初始止损校验；
- 后续卖出完全进入现有分层退出和统一有效止损流程。

## 10. 仓位与最小一手

二次确认目标仓位取以下结果中的最小值：

```text
正常策略目标仓位的 1/3
单笔风险预算允许仓位
账户及组合边界允许仓位
```

正常情况下按 100 股向下取整。

若结果不足 100 股，进入最小一手例外检查。允许提升到 100 股必须同时满足：

- 100 股金额不超过可用现金；
- 100 股风险金额不超过当前单笔风险预算；
- 不超过单票、行业、题材、无分类、总仓位和持仓数量上限；
- 不超过当前市场状态的开放风险上限；
- 当前不是 `RISK_OFF`；
- 全部健康、对账和交易控制仍允许买入。

```text
100 股风险金额
= 100 × (预计成交价 - 新有效止损价)
+ 手续费及预估滑点
```

任一条件不满足时记录明确拒绝原因，不得通过降低止损、虚构仓位比例或放宽风险预算强行买入。

## 11. 实际市场异常场景

### 11.1 开板后快速下杀

以下任一情况终止当天机会：

- 两次扫描之间下跌超过 1%；
- 跌破原计划入场位后继续走弱；
- 跌破当前重新计算的关键支撑；
- 流动性、买卖盘或成交质量异常。

价格降低本身不等于新的买入理由。

### 11.2 部分成交后回封

如果订单部分成交后重新封板：

- 保留已成交股份；
- 取消剩余未成交部分；
- 当天不补单、不加仓；
- 已成交股份按实际成本建立有效止损；
- T+1 导致当天无法卖出时，只记录止损触发和受限原因，不虚构卖出。

### 11.3 服务重启或数据中断

观察状态可以持久化，但恢复后必须校验：

- 同一交易日；
- 两个批次时间和 ID 都有效；
- 中间没有行情中断或未知提交；
- 行情、涨停价和交易状态仍新鲜。

无法确认时从第一次观察重新开始，不能拼接不连续证据。

### 11.4 除权除息和复权变化

原计划后发生除权、除息、送转或复权口径变化时，原价格锚点失效。必须重建完整普通交易计划，不得把复权变化误判为跳空。

### 11.5 停牌复牌

复牌股票不能使用停牌前计划直接进入二次确认。只有当前策略重新选中并生成当前时点依据后才能建立新机会。

### 11.6 不同板块涨跌停制度

使用当日证券级涨停价和板块元数据，兼容主板、创业板、科创板和北交所。ST、退市整理和特殊上市阶段继续按现有规则排除。

### 11.7 行情缺字段或陈旧

涨停价、报价时间、价格或交易状态不可确认时失败关闭，不使用固定百分比或上轮价格猜测。

### 11.8 同股票旧信号与新信号

- 原信号保持过期、拒绝或被替代状态；
- 二次确认使用新的稳定信号 ID；
- 新信号保存 `parent_signal_id`，但不修改原信号；
- 同股票每天最多一笔二次确认成交；
- 存在持仓、活动订单或未知提交时禁止新信号。

## 12. 持久化与增长治理

为支持直接模拟盘验证和错过行情的反事实复盘，交易账本新增事件级 `gap_reentry_opportunities` 表。每只股票、每个交易日、每个机会只插入一行，后续扫描更新同一行，不保存逐扫描无限历史。

建议字段：

- 机会 ID、交易日、股票代码；
- 原信号 ID、新信号 ID；
- 原入场、原止损、原始 R、二次最高价；
- 第一次开板时间和价格；
- 当前状态、确认次数、开板尝试次数；
- 新计划入场、止损、止盈和数量；
- 最终允许/拒绝原因；
- 委托、成交和终止结果；
- 创建、更新时间和策略/参数版本。

增长约束：

- 唯一键限制为每股、每日、机会 ID；
- 不按每轮扫描追加明细；
- 默认保留至少 2 年，年度容量复核；
- 按交易日、状态、股票代码建立必要索引；
- 纳入 SQLite 在线备份、恢复演练、完整性检查和 schema 指纹；
- 不新增 JSONL、永久逐轮文件或仓库根目录运行数据；
- 不保存 token、账户凭据、完整请求头或其他秘密。

正式 schema 版本和迁移步骤在实施计划中确定。迁移必须幂等，并在现有备份恢复流程中验证。

## 14. 通知

微信消息必须使用服务器当前时间，并区分观察和执行：

- `跳空二次确认：观察中`
- `涨停锁定：未追入`
- `开板后回封：确认已重置`
- `二次确认通过：已生成新买入计划`
- `最小一手风险超限：放弃买入`
- `执行前价格变化：取消信号`

观察通知按机会和状态变化去重。只有新执行契约已经发布时，才能使用“可买入”或“已提交买入”；锁板、观察和拒绝状态不得写成已经到达可执行买点。

## 15. 配置与权限

实施时增加独立开关，默认关闭。代码部署和服务器启用是两个独立动作：

- 代码进入服务器不自动启用；
- 启用只面向 JoinQuant 模拟盘；
- 修改服务器配置必须另获授权并保持现有 token 和秘密值不变；
- JoinQuant 网站模板更新必须保留 URL、token 和运行配置；
- Codex 自动审核员只读，不得批准、启用或改变该策略。

## 17. 状态与验收

状态必须严格区分：

- `planned`：本设计已确认，尚无代码；
- `implemented`：代码、迁移和测试完成；
- `deployed`：服务器与 JoinQuant 模板包含并启用该能力；
- `observed`：真实模拟交易日出现锁板、开板、回封、拒绝或成交证据；
- `validated`：满足样本、连续运行、执行安全和收益评估门槛。

建议验证门槛：

- 至少 20 个有效交易日；
- 无旧信号执行、重复下单、封板排队和风险绕过；
- 机会账本、订单、成交、持仓和通知可对账；
- 覆盖至少锁板、开板确认、回封重置、价格过高和风险拒绝等代表性分支；
- 对已执行与被拒绝机会计算 D+1/D+3/D+5/D+10、MFE、MAE、胜率、盈亏比和组合回撤；
- 与原策略比较新增收益、新增回撤和尾部损失；
- 严格历史数据可用后，补充至少 3 段 walk-forward 反前视回测；
- 最终 `validated` 由人工审核确认，不能由定时任务自动升级。

在事件数量不足时，即使运行满 20 个交易日也只能报告“持续观察，样本不足”，不能认定策略有效。

## 18. 实施顺序

用户确认本设计后，另行编写测试驱动实施计划。推荐批次：

1. 纯计算规则、状态机和原因码；
2. 机会账本及有界增长治理；
3. 服务器候选识别和新执行契约；
4. JoinQuant 执行前复核、部分成交和余单处理；
5. 通知、健康、复盘和反事实标签；
6. 全量回归、文档同步和本地提交；
7. 单独授权后推送、服务器部署、JoinQuant 模板更新和模拟盘启用；
8. 真实交易日观察和人工验证。
