# A 股买卖点与风控建议设计稿

## 目标

在现有 `a_share_strategy.py` 的选股/扫描基础上，增加一层独立的交易建议引擎，输出：

- 买入参考价
- 止损价
- 止盈价
- 仓位建议
- 交易模式标签：`short` 或 `mid`

这层逻辑不负责“选哪些股票”，只负责“如果要做这只票，怎么做更合理”。

## 设计原则

- 先判断交易是否成立，再计算买卖点
- 买点、止损、止盈、仓位要来自同一套风险框架
- 短线和中线共用同一接口，只切换参数
- 规则必须可回测、可解释、可调参
- 尽量依赖现有扫描结果，不额外引入复杂数据源

## 输入输出

### 输入

建议每只股票至少提供这些字段：

- `code`
- `name`
- `price`
- `high`
- `low`
- `open`
- `prev_close`
- `amount`
- `pct_chg`
- `turnover`
- `ma5`
- `ma10`
- `ma20`
- `ma30`
- `atr14`
- `support_level`
- `pressure_level`
- `market_state`
- `trend_state`

其中 `ma*`、`atr14`、`support_level`、`pressure_level` 可以先由历史行情计算，再逐步补齐。

### 输出

每只股票输出一个结构化结果：

- `mode`
- `entry_price`
- `stop_loss`
- `take_profit`
- `risk_per_share`
- `risk_reward`
- `position_pct`
- `reason`
- `confidence`

## 核心判断逻辑

### 1. 是否允许交易

先做硬过滤，避免在明显不合格的票上浪费计算：

- 价格过低
- 成交额过小
- 市场环境过弱
- 趋势明显破坏
- 涨跌停附近流动性异常

通过过滤后，才进入买点计算。

### 2. 模式识别

分成两种模式：

- `short`：突破型、事件驱动型、热点型
- `mid`：回踩确认型、趋势延续型

判定方式建议优先级如下：

1. 短线突破特征明显，进入 `short`
2. 否则如果趋势健康、回踩支撑有效，进入 `mid`
3. 两者都不满足，则不建议交易

### 3. 买点计算

#### `short`

买点参考：

- 突破平台上沿
- 放量站上 `ma10` 或 `ma20`
- 日内强势，接近压力位但未明显转弱

建议取值：

- `entry_price = max(突破位, 当日确认价)`

#### `mid`

买点参考：

- 回踩 `ma20` / `ma30`
- 回踩前高平台后止跌
- 量能缩减后重新转强

建议取值：

- `entry_price = 支撑位上方确认价`

## 风控计算

### 1. 止损

止损必须在买入前固定，不能追着亏损改。

#### `short`

推荐：

- `stop_loss = entry_price - 1.5~2.0 * ATR14`
- 或跌破突破位、平台下沿立即止损

#### `mid`

推荐：

- `stop_loss = 最近支撑位下方 1%~2%`
- 或 `entry_price - 2~3 * ATR14`

### 2. 止盈

止盈建议不要只用一个固定价，最好拆成两段：

- 第一段：`2R` 附近减仓
- 第二段：前高、压力位或趋势走坏时离场

其中 `R = entry_price - stop_loss`

### 3. 仓位

仓位按单笔最大亏损倒推：

- `max_loss_pct = 1%~2%`
- `position_value = account_value * max_loss_pct / (entry_price - stop_loss)`

这样止损越紧，仓位越大；止损越宽，仓位越小。

## 推荐模块拆分

### `risk_engine.py`

职责：

- 计算 `entry_price`
- 计算 `stop_loss`
- 计算 `take_profit`
- 计算 `position_pct`
- 输出交易建议结构

### `strategy_profile.py`

职责：

- 保存短线/中线参数
- 统一管理 ATR 倍数、止损比例、R 倍数、仓位上限

### `signal_adapter.py`

职责：

- 接现有 `a_share_strategy.py` 的扫描结果
- 把现有字段整理成风控引擎需要的输入

### `report_builder.py`

职责：

- 把建议输出成表格、Markdown、通知卡片
- 供命令行和企业微信复用

## 参数建议

### `short` 默认参数

- `stop_atr_mult = 1.8`
- `take_profit_r = 2.0`
- `max_loss_pct = 1.0%`
- `entry_confirm_pct = 0.3%`

### `mid` 默认参数

- `stop_atr_mult = 2.5`
- `take_profit_r = 3.0`
- `max_loss_pct = 1.5%`
- `entry_confirm_pct = 1.0%`

## 数据依赖

第一版尽量只依赖：

- 行情实时数据
- 日线历史数据
- 现有扫描结果

不强制引入复杂基本面模型，先把交易规则闭环做完整。

## 失败处理

如果某只股票缺少关键字段：

- 缺 `ATR`：退化为均线/支撑位止损
- 缺支撑位：只输出模式标签，不输出建议价
- 缺历史数据：跳过该票并记录原因

如果市场整体极弱：

- 降低仓位建议
- 提高止损严格度
- 或直接标记为不建议交易

## 测试方案

建议至少做四类测试：

1. 单元测试
   - 买点、止损、止盈、仓位公式正确
   - 短线和中线参数切换正确

2. 边界测试
   - 缺字段
   - 零波动
   - 极端高波动
   - 止损价高于买入价的异常情况

3. 回测样本
   - 选若干只历史票验证建议是否合理
   - 看 `R` 倍数和胜率是否能接受

4. 报表测试
   - 输出表格字段完整
   - Markdown/通知文本不截断

## 与现有代码的关系

现有 `a_share_strategy.py` 继续负责：

- 扫描股票
- 打分排序
- 新闻/情绪/行业增强

新增风控模块只在扫描结果后面接一层：

- 先筛出候选
- 再对候选生成交易建议

这样不会把选股逻辑和交易纪律搅成一团。

## 第一版范围

第一版只做：

- `short` / `mid` 两种模式
- 买入价、止损价、止盈价、仓位建议
- 与现有扫描结果的接入
- 命令行输出和 Markdown 输出

第一版不做：

- 自动下单
- 复杂机器学习预测
- 多周期联动决策
- 组合级资金管理

## 待确认项

- `atr14` 和均线是否直接复用现有历史数据接口计算
- 是否把仓位建议固定为百分比，还是同时输出股数
- 是否优先服务短线，还是默认中线优先
- 是否把这层逻辑接到企业微信通知里
