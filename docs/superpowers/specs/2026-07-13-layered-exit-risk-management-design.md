# 分层退出、买入与动态风险管理设计

更新日期：2026-07-14

> 主文档：`docs/project_roadmap.md`。本文是当前买入、卖出和风险控制的权威专项设计；实施与部署观察顺序见 `docs/superpowers/plans/2026-07-13-layered-exit-risk-management.md`。状态冲突时以主文档为准。

## 1. 状态边界

当前本地工作区已 `implemented`：分层退出、全持仓硬止损、持仓周期、执行安全、真实组合买入风控、可交易性过滤、市场状态滞后和退出冷却。它们尚未提交、推送、部署、观察或验证；服务器最近确认仍为 SQLite schema version 1，不能把本地 schema version 5 或网站模板 `2026-07-13.3-risk-controls` 写成服务器事实。

状态含义固定为：

- `planned`：文档设计，代码未完成。
- `implemented`：本地代码和测试完成。
- `deployed`：服务器和 JoinQuant 已运行对应版本。
- `observed`：真实模拟盘交易日已产生证据。
- `validated`：达到连续天数、执行一致性和样本门槛并经人工确认。

## 2. 职责与数据流

服务器负责扫描、评分、计算交易建议、组合风险和最终信号；JoinQuant 负责模拟盘实际下单、T+1、停牌、涨跌停、100 股交易单位和撮合。账户、持仓、委托与成交由 JoinQuant 回传服务器，服务器再更新 SQLite、健康报告和通知。

```text
全市场行情 + 策略候选 + JoinQuant同步账户/持仓/委托
→ 买入与退出纯规则
→ 组合风险和可交易性检查
→ 同股唯一最终动作
→ JSON schema 1 + SQLite账本
→ JoinQuant目标数量下单
→ 委托/成交/快照回传与对账
```

## 3. 买入建议

### 3.1 模式

- `short`：突破、热点或事件驱动，使用更紧的时间窗口和风险预算。
- `mid`：回踩确认或趋势延续，允许更宽的波动空间和更长持有期。

模式只决定参数族，不绕过统一组合风控和可交易性检查。输入至少包含股票代码、价格、开高低收、成交额、换手率、均线、ATR14、支撑/压力、市场状态和趋势状态。

### 3.2 入场

- `short`：突破位或确认价取更高者，超过计划入场价 `min(2%, 0.5ATR/entry)` 时不追买。
- `mid`：支撑位、`ma20/ma30` 或前高平台回踩确认后入场。
- 当前价未达到确认位、接近涨停、停牌、ST/*ST、退市整理、特殊上市阶段、流动性不足、行情陈旧或价格异常时拒绝新买。

JoinQuant 下单前必须以平台当前价格和可用现金再次复核；服务器候选结果不是无条件下单指令。

## 4. 初始止损、R 与目标仓位

建仓时冻结初始止损，持仓后不得向下放宽：

```text
R = 入场价 - 冻结初始止损价
止损距离比例 = R / 入场价
目标持仓金额 = 账户总资产 × 单笔账户风险比例 ÷ 止损距离比例
```

参考基线：

| 类型 | 初始止损 | 最大止损距离 | 单笔账户风险 | 首段止盈 | 剩余仓位保护 |
| --- | --- | ---: | ---: | ---: | --- |
| 主板低波动 | 支撑下方与 `1.5-1.8 ATR` 取更宽者 | 6% | 0.5%-0.8% | `+2R` 降至初始半仓 | 最高可信价减 `2 ATR` |
| 主板活跃股 | 支撑下方与 `1.8-2.2 ATR` 取更宽者 | 7% | 0.5% | `+2R` 降至初始半仓 | 最高可信价减 `2-2.5 ATR` |
| 创业板/科创板 | 支撑下方与 `2.0-2.5 ATR` 取更宽者 | 9% | 0.3%-0.5% | `+2R` 降至初始半仓 | 最高可信价减 `2.5-3 ATR` |

推荐值是冻结观察基线，不因单日盈亏改变。后续参数复核必须遵守 `docs/superpowers/specs/2026-07-14-semi-automatic-parameter-review-design.md`。

## 5. 全持仓退出决策

退出检查覆盖全部 JoinQuant 同步持仓，不要求股票进入当轮候选池：

1. 优先使用当轮全市场实时价。
2. 实时价格缺失或无效时回退到同步持仓最近可信现价。
3. 只有活动持仓、数量大于零、代码和价格有效时参与判断。
4. 历史持仓缺少完整周期时，只沿用已有且大于零的固定 `stop_price`，不猜测 ATR、R、最高价或持仓天数。
5. 同股票已有买入候选时，退出动作覆盖买入，保证同轮唯一动作。

退出优先级固定为：

```text
硬止损
> 市场风险退出
> 首段止盈后的移动止盈
> +2R首段止盈
> 时间止损
> 买入或继续持有
```

执行规则：

- 现价低于或等于冻结硬止损：目标数量为 0。
- 首次达到 `+2R`：目标持仓降至当前周期初始数量的 50%，并遵守 100 股单位。
- 首段止盈已确认成交后，现价低于或等于只能上移的移动止损：目标数量为 0。
- `short` 持有 3 个交易日未达到 `+0.5R`：时间止损。
- `mid` 持有 10 个交易日未达到 `+1R`：时间止损。

止盈阶段只能由已确认成交或同步持仓数量达到目标推进，不能因“曾生成信号”推进。

## 6. 持仓周期与执行状态

SQLite 本地 schema version 5 包含：

- `position_cycles`：买入信号、首次成交、加权成本、初始数量、冻结止损、初始 R、ATR、建仓市场状态、最高可信价和止盈阶段。
- `order_events`：委托 ID、状态、数量、成交量、失败原因和回调幂等。
- `exit_intents`：目标数量、剩余目标、最后对账时间和未完成退出原因。
- `trade_cooldowns`：退出原因、冷却到期和再买约束。

每只股票最多一个活动周期；清仓后关闭，再次建仓创建新周期。加仓更新加权成本但不得放宽冻结止损。重复快照、回调和重启不得串用旧周期或重复推进状态。

委托状态至少区分 `submitted / partial / filled / cancelled / rejected / T+1 / suspended / limit_down`。未完成委托存在时不得重复下单；部分成交或撤单后按剩余目标重算；不可卖或跌停不能把退出标为完成。

## 7. 组合买入风控

最终买入检查使用真实账户可用现金、总资产、已有持仓、未完成买单冻结资金、当日新增仓位、真实成交换手和账户损益。

- 组合开放风险包含活动持仓和未完成买单风险。
- 默认开放风险上限：`NORMAL=4%`、`CAUTION=2%`、`RISK_OFF=0%`。
- 单行业目标市值不超过 25%，同题材不超过 20%，分类缺失时单票不超过 10%。
- 候选按最终评分和风险收益质量排序，依次消耗现金与风险容量；缩成 100 股单位后不足一手则拒绝。
- 连续亏损交易日、账户回撤、当日新增持仓、订单数和换手率进入强制层。

## 8. 市场状态与冷却

- `NORMAL → CAUTION → RISK_OFF` 至少连续 2 轮确认。
- 风险恢复至少连续 3 轮确认，状态持久化且重启不丢失。
- `RISK_OFF` 禁止新买；`CAUTION` 只接受高分并将风险预算减半。
- 市场状态不得下移初始止损；只能收紧盈利保护或减少新增风险。
- 硬止损、时间止损、正常止盈和人工退出使用不同冷却期限；冷却结束和新信号同时成立才能再买。

## 9. JoinQuant 契约与安全降级

信号 JSON 保持 schema version 1，并使用可选 `target_qty` 表达部分卖出。JoinQuant 模板 `2026-07-13.3-risk-controls` 使用 `order_target` 执行目标数量，检查可卖数量、冻结数量、未完成委托和当前价格。

安全降级原则：

- 账本不可读时停止新买，但不能阻止合法硬止损和减仓。
- 缺 ATR、R、建仓日或周期状态时，不启用推导型移动止盈和时间止损，只保留可信固定硬止损。
- 价格为 0、时间戳陈旧或偏离最近可信价超过阈值时暂停新动作并报警，不以异常价触发卖出。
- T+1、停牌、跌停和临时拒单保留退出意图，直到成交或持仓归零。

## 10. 存储与验证

持仓周期、委托事件、退出意图和冷却属于长期审计数据，不按日志清理。`TradingStore.backup_to`、恢复副本读取和 `PRAGMA integrity_check` 已在本地 `implemented`；自动定时备份、7日/4周/12月轮转和服务器恢复演练仍为 `planned`。

部署与验证必须分层：本地测试完成只代表 `implemented`；服务器与 JoinQuant 模板同步后是 `deployed`；首个有效交易日逐笔一致才是 `observed`；连续 3 日执行安全和至少 20 日策略观察达到门槛后，相关能力才能人工标为 `validated`。
